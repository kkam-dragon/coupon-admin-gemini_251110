# Repository Agent Instructions

1. **문서 참조 필수**:  `MyDocuments/`의 내용을 확인하여 아키텍처 및 백앤드 기술사항을 `docs/` 폴더에 만들고, 코드 또는 문서를 수정하기 전에 `docs/` 및 `MyDocuments/` 폴더의 관련 문서를 확인하고 요구사항과 일치하도록 반영하세요.
2. **언어 선택**: 백엔드 구현은 기본적으로 Python 3.11 + FastAPI 스택을 사용합니다. 다른 언어를 제안하거나 사용할 경우 제품 담당자의 승인을 문서화해야 합니다.
3. **보안 준수**: 개인정보 및 쿠폰번호는 암호화/마스킹 정책을 만들고 반드시 따릅니다. 예제나 로그에 실제 값을 노출하지 마세요.
4. **문서 업데이트**: 요구사항이나 설계가 변경되면 관련 문서를 먼저 업데이트하고, 커밋 메시지에 `[docs]` 태그를 포함하세요.
5. **테스트 보고**: 코드 변경 시 실행한 테스트 명령과 결과를 PR/최종 응답에 명시하세요.
6. **GEMINI확장**: 디렉터리별 세부 지침이 필요한 경우 해당 디렉터리에 추가 `GEMINI.md`를 생성하여 보강할 수 있습니다.
7. **모든 문서 갱신 시 백업**: 모든 문서는 갱신될 시 반드시 백업을 해놓고 갱신을 합니다. 백업 형식은 `원본파일명-YYYYMMDD-seq.원본확장자` 형태로 이전 내용을 보존합니다.(예: 원본파일명.md > 원본파일명-20251109-1.md)
8. ***대화 언어**: 대화 언어는 반드시 한글로만 해야 합니다.
